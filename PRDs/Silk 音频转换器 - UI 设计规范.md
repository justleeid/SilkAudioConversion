# Silk 音频转换器 - UI 设计规范（精简版・仅原则 / 规范 / 约束）

**版本：v2.1**

**设计定位：克制、精致、现代工具类高级感**

------

## 1. 设计理念与原则

- **设计哲学**：少即是多，克制即高级；秩序 × 呼吸感 × 材质感

- 核心原则

  ：

  1. **极简克制**：无冗余视觉、无过度装饰
  2. **视觉统一**：色彩、圆角、阴影、间距全局一致
  3. **精致细节**：微动效、分层阴影、柔和渐变
  4. **流畅交互**：状态过渡自然、反馈清晰、无跳变
  5. **清晰层级**：信息主次明确、操作路径最短

- **参考风格**：Linear / Vercel / Apple / Raycast

------

## 2. 色彩规范

- **主色调**：蓝色系，强调色仅用于关键交互

- **模式**：支持浅色 / 深色双模式，自动跟随系统

- 规则

  ：

  - 遵循 **60-30-10-5** 色彩比例
  - 高饱和色占比 ≤5%
  - 禁用纯黑 / 纯白，全部做柔化处理
  - 状态不依赖颜色，需搭配图标 / 文字

------

## 3. 字体与排版规范

- **字体家族**：系统无衬线优先，中文优雅、英文等宽

- **层级**：标题→副标题→正文→辅助文字 四级结构

- 规则

  ：

  - 字重、间距、行高统一规范
  - 数字 / 英文等宽对齐
  - 长文本溢出统一省略号处理
  - 响应式自动缩放字号

------

## 4. 布局与间距规范

- 布局策略

  ：

  - **PC 端**：三栏固定等高布局，全屏利用横向空间
  - **移动端**：单列流式布局，核心区域固定
  - 所有长列表**内部滚动**，不撑开页面

- **栅格**：12/8/4 列响应式栅格

- **间距**：以 **8px 为基准**，模块间距统一层级

- 约束

  ：

  - 内容密度 ≤60%，保证呼吸感
  - 触摸目标 ≥44px（移动端）

------

## 5. 组件规范

- **卡片**：统一圆角、轻阴影、分层悬浮感

- **按钮**：主按钮渐变质感，次按钮克制简洁

- **表单**：焦点反馈清晰、状态统一

- **上传区**：拖拽友好、状态明确、微动效

- **列表项**：状态可视化、左侧色条标识、悬停反馈

- 通用约束

  ：

  - 圆角风格统一
  - 阴影分层（sm /md/lg）
  - 交互必须有 hover /active 状态

------

## 6. 动效规范

- **原则**：适度、克制、物理惯性

- **曲线**：标准 easing 统一，无生硬跳动

- 场景

  ：

  - 悬停：上浮 + 阴影增强
  - 点击：轻微按压反馈
  - 切换：淡入淡出 + 滑入
  - 完成：弹性正向反馈

- **约束**：可关闭动效，兼顾可访问性

------

## 7. 响应式与适配规范

- **桌面端**：三栏并行，固定高度，内部滚动

- **平板端**：双列自适应

- **移动端**：单列、底部固定操作栏

- 适配目标

  ：

  - 横竖屏正常展示
  - 核心操作路径不变
  - 视觉比例统一

------

## 8. 深色模式规范

- 背景：深灰非纯黑，低饱和度更护眼
- 卡片：磨砂玻璃质感 + 模糊效果
- 文字：暖白非纯白，降低刺眼感
- 强调色：降低饱和度，提升舒适度
- 自动 / 手动切换，过渡平滑

------

## 9. 可访问性（A11y）规范

- 支持键盘完整导航
- 屏幕阅读器适配（aria-label）
- 对比度符合 WCAG AA 标准
- 状态不只用颜色表达
- 提供减少动画选项

------

## 10. 工程约束（必守）

- 与后端接口、字段、状态枚举**完全对齐**
- 上传格式、参数范围、限制文案保持一致
- UI 服从技术规范，不做无法实现的设计
- 所有交互可闭环、可验证、可回溯

------

## 11. 技术实现映射

### 11.2 样式方案：Tailwind CSS

**用途范围：**
- 页面整体布局（flex / grid / 三栏结构）
- 间距系统（p-/m-/gap-，基准 8px → 1 = 0.25rem → 例如 gap-2 = 8px）
- 响应式断点（sm: / md: / lg: / xl:）
- 颜色与暗色模式（dark: 前缀，配合 Naive UI 的暗色主题）
- 文字层级（text-sm / text-base / text-lg / font-medium / font-semibold）

**与 Naive UI 的分工：**
- Tailwind → 布局、间距、颜色、响应式
- Naive UI → 交互组件（表单、表格、弹窗、通知）
- `<style scoped>` → 仅在 Tailwind 无法覆盖的少数场景使用（自定义动画、滚动条样式、组件覆写）

### 11.3 暗色模式

Naive UI 内置暗色模式，通过 `NConfigProvider` 一行切换：

```vue
<n-config-provider :theme="isDark ? darkTheme : lightTheme">
  <n-message-provider>
    <App />
  </n-message-provider>
</n-config-provider>
```

- 浅色 / 深色双模式，自动跟随系统 `prefers-color-scheme`
- Tailwind 通过 `dark:` 前缀同步适配
- 背景：深灰非纯黑（`dark:bg-gray-900`），文字暖白（`dark:text-gray-100`）

### 11.4 页面布局方案

**桌面端（≥1024px）：三栏固定高度，全屏利用**

```
┌────────────────┬──────────┬──────────────┐
│  栏1: 文件输入  │ 栏2: 设置 │ 栏3: 任务结果  │
│  flex: 3       │ flex: 2  │ flex: 3      │
│  h-full        │ h-full   │ h-full       │
│  overflow-y:auto│          │ overflow-y:auto│
└────────────────┴──────────┴──────────────┘
```

**平板端（768-1023px）：双列自适应**
**移动端（<768px）：单列，底部固定操作按钮**

Tailwind 实现：
```html
<div class="h-screen flex flex-col lg:flex-row gap-4 p-4">
  <!-- 栏1: 文件输入区 -->
  <div class="flex-3 min-w-0 overflow-y-auto">...</div>
  <!-- 栏2: 转换设置区 -->
  <div class="flex-2 min-w-0 shrink-0">...</div>
  <!-- 栏3: 任务结果区 -->
  <div class="flex-3 min-w-0 overflow-y-auto">...</div>
</div>
```
